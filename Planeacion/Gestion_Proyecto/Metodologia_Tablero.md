# 📋 Gestión del Proyecto Ágil

Para convertir nuestras Historias de Usuario (descritas en el documento de Word) a código, utilizaremos un sistema de gestión visual de tarjetas conocido como **Tablero Kanban**.

## 💡 ¿Cómo funciona el tablero?
Dividiremos el progreso del trabajo en columnas sencillas:
1. **Backlog (Pendientes):** Lista de todas las historias de usuario que faltan por programar.
2. **In Progress (En curso):** Lo que cada programador está haciendo activamente *hoy*.
3. **Review (En revisión):** Código terminado esperando aprobación del líder técnico en GitHub (Pull Request).
4. **Done (Completado):** Funcionalidad programada, revisada y funcional.

## 🛠️ Herramienta Sugerida (GitHub Projects)
Ya que tienen el código alojado en GitHub, la forma más fácil y profesional de organizarse es usando **GitHub Projects**.

* **Pasos rápidos para crearlo:**
  1. Entra a tu repositorio oficial en GitHub.
  2. Haz clic en la pestaña superior que dice **"Projects"**.
  3. Haz clic en el botón verde **"Link a project"** o en **"New project"**.
  4. Selecciona el template tipo **"Board"** (eso creará las columnas por ti).
  5. ¡Listo! Ya puedes darle al botón de "+" abajo de tus columnas para escribir el título de tus Historias de Usuario.
